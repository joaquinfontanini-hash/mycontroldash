IMPORTANTE

Quiero un sistema de acceso sólido, con autenticación fuerte, control de usuarios, roles claros, trazabilidad y capacidad de administración total por parte del superusuario.

==================================================

Quiero que el acceso al sistema sea mucho más seguro.

REQUISITOS

- Para áreas críticas, contemplar step-up authentication o revalidación de identidad
- Mantener sesiones seguras
- No guardar credenciales sensibles en frontend
- Manejar backend session/token de manera segura
- Implementar:
  - expiración de sesión
  - cierre de sesión seguro
  - invalidación de sesiones
  - protección contra acceso indebido
- Evitar autenticación insegura basada solo en email sin validación real
- Preparar la solución para producción, con variables de entorno y configuración robusta

IMPORTANTE

- Priorizar una solución razonable, mantenible y sólida

==================================================

Necesito un esquema de roles bien definido.

ROLES MÍNIMOS
- super_admin
- admin
- editor
- viewer

REQUISITOS

- El super_admin debe tener acceso total a todo el sistema
- El super_admin debe poder:
  - ver todos los módulos
  - habilitar o deshabilitar módulos
  - bloquear usuarios
  - desbloquear usuarios
  - restablecer contraseñas o reinicios de acceso según el sistema implementado
  - cambiar roles
  - activar o desactivar cuentas
  - cerrar sesiones activas de otros usuarios si hace falta
  - ver logs de actividad
  - gestionar permisos globales
- El admin debe tener permisos altos pero no superiores al super_admin
- El editor debe poder operar donde corresponda
- El viewer solo debe ver lo habilitado
