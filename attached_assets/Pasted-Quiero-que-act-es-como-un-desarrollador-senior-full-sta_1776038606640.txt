Quiero que actúes como un desarrollador senior full stack, experto en seguridad web, autenticación, control de accesos, Cloudflare Pages, Cloudflare Workers, D1 y aplicaciones web administrativas.

Necesito que construyas una aplicación web profesional, auditada, validada y pensada para producción, no un prototipo visual vacío.

OBJETIVO GENERAL
Crear un panel web con login seguro, gestión de usuarios y suscripción de nuevos usuarios con aprobación manual por parte del administrador.

REQUISITOS FUNCIONALES OBLIGATORIOS

1) AUTENTICACIÓN
- Debe existir una pantalla de login moderna, clara y profesional.
- Debe permitir iniciar sesión con:
  - email + contraseña
- Preparar la estructura para futura integración con Google OAuth, pero por ahora dejarlo opcional o desacoplado.
- El sistema NO debe guardar contraseñas en texto plano.
- Las contraseñas deben almacenarse con hash seguro.
- Debe existir cierre de sesión.
- Debe existir control de sesión.
- Debe evitar acceso a páginas internas sin autenticación.

2) ROLES DE USUARIO
Debe haber al menos estos roles:
- superadmin
- admin
- user

Permisos:
- superadmin:
  - acceso total
  - puede crear admins y users
  - puede aprobar o rechazar solicitudes de registro
  - puede activar o desactivar usuarios
  - puede editar usuarios
  - puede eliminar usuarios
- admin:
  - puede crear usuarios comunes
  - puede ver solicitudes pendientes
  - puede aprobar o rechazar usuarios si así se configura
  - no puede modificar al superadmin
- user:
  - solo puede ingresar al panel que le corresponda
  - sin acceso a administración

3) ALTA MANUAL DE USUARIOS DESDE ADMIN
- Desde el panel de administración debe existir una sección “Usuarios”.
- El admin o superadmin debe poder:
  - crear usuario manualmente
  - definir nombre
  - definir email
  - definir rol
  - asignar contraseña temporal o enviar invitación
  - activar o desactivar acceso
  - editar datos básicos
- Validar que no se repitan emails.

4) SUSCRIPCIÓN PÚBLICA CON APROBACIÓN
- Debe existir una pantalla pública de “Registrarse” o “Solicitar acceso”.
- Cualquier persona puede completar:
  - nombre
  - apellido
  - email
  - contraseña
  - confirmación de contraseña
  - opcional: motivo o comentario de solicitud
- Cuando alguien se registra:
  - NO debe quedar habilitado automáticamente
  - debe quedar en estado “pendiente”
  - el admin debe ver esa solicitud en una bandeja de aprobaciones
- El admin o superadmin debe poder:
  - aprobar solicitud
  - rechazar solicitud
  - ver fecha de solicitud
  - ver datos del solicitante
- Si se aprueba:
  - el usuario pasa a estado “activo”
  - ya puede iniciar sesión
- Si se rechaza:
  - el usuario queda como rechazado o eliminado según el diseño elegido

5) ESTADOS DE USUARIO
Cada usuario o solicitud debe poder tener estados:
- pending
- active
- suspended
- rejected

6) PANEL DE ADMINISTRACIÓN
El dashboard admin debe tener como mínimo:
- resumen de usuarios totales
- usuarios activos
- usuarios pendientes de aprobación
- usuarios suspendidos
- listado de usuarios
- listado de solicitudes pendientes
- acciones rápidas:
  - aprobar
  - rechazar
  - editar
  - suspender
  - reactivar
  - eliminar

7) EXPERIENCIA DE USUARIO
- Diseño moderno, profesional y limpio
- Responsive
- Preparado para desktop y mobile
- Interfaz clara, sin errores visuales
- Mensajes de éxito y error bien hechos
- Formularios con validaciones visibles
- Buen manejo de estados de carga
- Evitar pantallas rotas o vacías
