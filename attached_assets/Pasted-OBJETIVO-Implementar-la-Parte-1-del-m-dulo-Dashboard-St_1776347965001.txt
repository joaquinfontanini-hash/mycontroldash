OBJETIVO:
Implementar la Parte 1 del módulo "Dashboard Studio" dentro del sistema existente.

Esta Parte 1 debe dejar construida la base estructural, persistencia, catálogo, plantillas, motor de generación inteligente v1 y APIs necesarias para crear dashboards dinámicos desde prompt, plantilla o wizard, pero SIN construir todavía el builder visual avanzado ni la lógica de snapshots optimizados.

Actuá como arquitecto senior, backend lead, frontend lead y auditor técnico. Reutilizá la arquitectura existente. No rompas módulos actuales.

MOTOR DE GENERACIÓN INTELIGENTE V1
==================================================

Crear un servicio de generación desde prompt que funcione sin LLM externo.

Flujo:
1. normalizar prompt
2. detectar dominio principal
3. detectar dominios secundarios
4. detectar entidades
5. detectar intención
6. detectar widgets razonables
7. detectar filtros sugeridos
8. generar metadata del dashboard
9. generar layouts default
10. generar JSON persistible
11. registrar run en dashboard_runs

Heurísticas mínimas:
- clientes + vencimientos → KPI + semáforo + tabla + alertas
- finanzas + gastos → resumen + categorías + movimientos + presupuestos
- noticias + economía → feed + top impacto + alertas
- tareas + equipo → pendientes + ranking + actividad
- dirección ejecutiva → KPIs + alertas + resumen inteligente + actividad

No inventar data sources inexistentes.
No inventar widgets incompatibles.
Si no se puede inferir perfecto, generar una versión mínima usable.
