1) CHAT INTERNO ENTRE USUARIOS
==================================================

Quiero un sistema de chat interno dentro de la app.

REQUISITOS GENERALES
- Debe estar integrado con el sistema actual de usuarios
- Solo usuarios autenticados pueden acceder
- Debe respetar roles y permisos si corresponde
- Debe verse como parte nativa del dashboard, no como algo externo
- Debe ser rápido, claro y usable

FUNCIONALIDADES
- Lista de conversaciones
- Posibilidad de iniciar conversación con otro usuario
- Vista de chat con mensajes
- Campo para escribir y enviar mensaje
- Timestamp por mensaje
- Mostrar quién envió cada mensaje
- Estado visual básico:
  - enviado
  - leído si es viable en esta etapa
- Búsqueda de usuarios para iniciar chat
- Notificación o badge de mensajes no leídos
- Integración con campana o alertas si tiene sentido

TIPO DE CHAT
- Primero implementar chat directo 1 a 1 entre usuarios
- Dejar preparada arquitectura para grupos futuros
- Si da el alcance, agregar conversaciones grupales como base, pero priorizar que funcione bien el chat individual

PERSISTENCIA
- Si el sistema sigue con localStorage o almacenamiento local en esta etapa:
  - persistir conversaciones y mensajes de forma ordenada
- Dejar arquitectura lista para backend o websocket futuro

UI/UX
- Panel de conversaciones
- Panel de mensajes
- Vista moderna tipo chat interno profesional
- Buen espaciado
- Avatares o iniciales
- Diferenciar mensajes propios y ajenos
- Responsive
- No truncar mensajes
- Soportar textos largos sin romper el layout

SEGURIDAD Y CONTROL
- Solo ver chats donde el usuario participa
- No permitir acceder a conversaciones ajenas por URL o manipulación del estado
- Validar remitente y destinatario
- Integrar con usuario autenticado actual

==================================================

Quiero que cada usuario tenga una tarjeta de contacto visible dentro del sistema.

DATOS DE LA TARJETA
- nombre
- apellido si existe
- rol
- teléfono
- email
- foto o avatar
- estado del usuario:
  - activo
  - bloqueado
  - inactivo
- opcional:
  - área
  - observaciones
  - último acceso

REQUISITOS
- Crear una vista o sección de contactos/usuarios
- Mostrar tarjetas de usuario visualmente prolijas
- Cada tarjeta debe tener:
  - foto o avatar
  - nombre completo
  - email
  - teléfono
  - rol
  - acción para ver perfil o iniciar chat
- Si no hay foto, usar avatar con iniciales elegante
- Diseño moderno y consistente con el resto del dashboard
- Responsive
- Sin componentes truncos

FUNCIONALIDAD

- Super admin o admin deben poder modificar teléfono, foto y datos del usuario si aplica
- El propio usuario debería poder completar o editar su foto y teléfono si la arquitectura actual lo permite
